VXLAN【2~3层技术】

Virtual Extensible LAN是一种网络虚拟化技术，用于扩展现有的以太网规模，并提供跨物理网络的虚拟网络隔离。以下是VXLAN的应用和特点：

应用：

  1. 多租户网络：VXLAN允许在共享基础设施上创建多个虚拟网络，每个虚拟网络都可以独立配置和管理，实现租户之间的隔离。
  2. 数据中心互连：VXLAN可用于连接不同数据中心之间的虚拟网络，提供跨数据中心的虚拟机迁移和高可用性。
  3. 跨主机通信：VXLAN可以在不同物理主机上的虚拟机之间提供透明的二层通信，扩展了虚拟机之间的网络范围。
  4. 跨子网通信：VXLAN可以将位于不同子网的虚拟机连接到同一个虚拟网络中，消除了传统以太网的子网限制。

特点：

  1. 扩展性：VXLAN使用24位的虚拟网络标识符（VNI）来扩展以太网的规模，可以支持数百万个虚拟网络。
  2. 隔离性：VXLAN提供虚拟网络之间的隔离，每个虚拟网络都有独立的标识符和隧道，避免了冲突和干扰。
  3. 兼容性：VXLAN基于标准的UDP封装，可以在现有网络设备上轻松部署，而无需进行大规模的网络改造。
  4. 灵活性：VXLAN可以与其他网络虚拟化技术（如VLAN、GRE等）结合使用，以满足不同场景下的需求。
  5. 路由支持：VXLAN可以与传统IP路由器集成，支持在虚拟网络之间进行跨子网的路由。
  6. 多厂商支持：VXLAN已经成为一项开放的行业标准，在多个网络设备厂商和云服务提供商中得到广泛支持。

SDN【2~3、3~4层技术】

Software-Defined Networking是一种网络架构和管理方法，通过将网络控制平面与数据转发平面分离，并使用集中式控制器对网络进行编程和管理。以下是SDN的应用和特点：

应用：

  1. 网络虚拟化：SDN可以实现网络的虚拟化，将物理网络资源划分为多个虚拟网络，每个虚拟网络可以独立配置和管理，实现多租户隔离。
  2. 动态流量工程：SDN可以根据网络流量状况和业务需求，动态地调整网络中的流量路径和带宽分配，以提高网络性能和优化资源利用率。
  3. 随需应变的服务交付：SDN的编程性质使得网络服务可以根据需求进行快速部署和配置，实现灵活的服务交付和管理。
  4. 数据中心网络：SDN可以用于构建大规模的数据中心网络，提供高可用性、灵活性和自动化管理。
  5. 软件定义的广域网（SD-WAN）：SDN可以应用于广域网，通过集中式控制和智能路由，优化广域网连接和管理，提供更好的性能和可靠性。

特点：

  1. 灵活性和可编程性：SDN的核心思想是将网络控制逻辑集中到一个可编程的控制器中，使网络可以根据需求进行快速配置和编程。
  2. 集中式控制：SDN使用集中式控制器对网络进行管理和编程，从而实现对整个网络的全局视图和控制。
  3. 分离的控制平面和数据平面：SDN将网络控制平面与数据转发平面分离，使控制决策与数据转发分开，提高了网络的灵活性和可扩展性。
  4. 开放性和标准化：SDN采用开放的协议和接口，使得不同厂商的设备和控制器可以互操作，促进了网络设备和应用的创新和发展。
  5. 自动化和集中管理：SDN的集中式控制和编程能力使得网络的配置、监控和故障排除可以自动化和集中管理，提高了网络的可管理性和运维效率。
  6. 软件定义的安全：SDN可以与安全策略和应用集成，实现对网络流量的细粒度控制和安全策略的动态更新，提高网络的安全性和防御能力。

OSPF【3层协议】

Open Shortest Path First是一种内部网关协议（IGP），主要用于内部网络的路由选择和转发。对于外部网络和边界路由，通常会使用外部网关协议（EGP）如BGP（Border Gateway Protocol）来实现跨自治系统的路由。

应用：

  1. 内部网络路由：OSPF用于在企业网络中实现内部路由，使得数据包可以在网络中的各个子网之间进行转发。
  2. 多路径路由：OSPF支持多路径路由，可以根据网络的拓扑结构和路由成本选择最佳的路径进行数据转发，提高网络的可靠性和性能。
  3. 动态路由调整：OSPF可以根据网络拓扑的变化，自动调整路由表，选择新的最佳路径，以适应网络的动态变化。
  4. 跨自治系统（AS）路由：OSPF也可以用于连接不同自治系统（AS）之间的路由器，实现跨域的路由选择和转发。
  5. QoS支持：OSPF支持QoS（Quality of Service）机制，可以根据服务质量要求对不同类型的流量进行优先级和带宽的分配。

特点：

  1. 开放性和标准化：OSPF是一种开放的协议，被广泛应用于不同厂商的路由器和网络设备中，并且具有良好的互操作性。
  2. 动态路由选择：OSPF使用链路状态信息，通过交换路由信息和链路状态更新，计算最短路径并选择最佳的路由。
  3. 支持分层设计：OSPF采用分层的设计结构，将网络划分为不同的区域（区域），减少网络中的路由更新和计算开销，提高网络的可扩展性和性能。
  4. 支持负载均衡：OSPF可以根据网络的拓扑结构和路由成本，将流量分布在多条路径上，实现负载均衡，提高网络的利用率和容错性。
  5. 高度可靠性：OSPF具有快速的收敛性和故障检测能力，当网络中发生故障或链路状态变化时，它可以快速调整路由表，确保网络的稳定性和可靠性。
  6. 灵活的策略控制：OSPF支持灵活的路由策略控制，可以根据需求对路由进行策略调整，实现灵活的网络管理和控制。

BGP【5层协议，但运行在4层】

应用：

  1. 互联网路由：BGP主要用于互联网中的路由器之间的通信，它负责交换和学习不同自治系统（AS）之间的路由信息，实现跨自治系统的路由选择和转发。
  2. 互联网连接：BGP被广泛应用于互联网服务提供商（ISP）之间的连接，通过交换路由信息，使得ISP可以相互通信和转发数据流量。
  3. 云服务提供商：BGP还被云服务提供商用于实现与客户之间的互联和路由管理，使得云服务的虚拟机和资源能够通过BGP协议与客户网络进行通信。

特点：

  1. 路径向量协议：BGP是一种路径向量协议，它在路由信息中携带了路径信息，不仅考虑最短路径，还考虑了其他因素如策略、自治系统路径长度等，以实现更复杂的路由选择。
  2. 高度可靠性：BGP设计了多种机制来保证路由的可靠性和稳定性，包括路由策略、路由过滤、路由聚合、路由重分发等，以应对网络拓扑变化和故障情况。
  3. 跨自治系统路由：BGP是一种跨自治系统的协议，它允许不同自治系统之间的路由器进行通信和路由学习，实现了互联网中不同网络的互通。
  4. 策略控制：BGP提供了灵活的策略控制机制，允许网络管理员定义和调整路由策略，以满足特定的需求，如流量工程、策略路由、防火墙策略等。
  5. 拓扑聚合：BGP支持路由聚合机制，可以将多个具体的路由聚合成更一般的路由，减少路由表的规模，提高路由表的可扩展性和效率。
  6. 安全机制：BGP引入了安全机制，如BGP MD5认证、路由过滤等，用于保护路由信息的完整性和防止恶意攻击，提高网络的安全性。

IS-IS【3层协议】

IS-IS（Intermediate System to Intermediate System）是一种内部网关协议（IGP），用于在自治系统内部进行路由选择和转发。下面是IS-IS的应用和特点：

应用：

  1. ISP网络：IS-IS广泛应用于Internet服务提供商（ISP）的网络中，用于实现自治系统（AS）内部的路由选择和转发。
  2. 企业网络：IS-IS也可以用于大型企业网络中，特别是在具有复杂拓扑结构和多个分支机构的网络中。
  3. 光纤通信网络：IS-IS被用于光纤通信网络中的光传输网（OTN），实现多层次、多技术的路由控制。
  4. 移动网络：IS-IS在移动网络中也有应用，特别是在移动核心网（Mobile Core Network）中，用于实现移动用户的路由控制和转发。

特点：

  1. 策略无关：IS-IS是一种策略无关的协议，它主要关注网络的拓扑结构和链路状态，而不关注具体的路由策略。
  2. 基于链路状态：IS-IS使用链路状态信息，通过交换链路状态更新和链路状态数据库（LSDB），计算最短路径并选择最佳的路由。
  3. 分层设计：IS-IS采用分层设计结构，将网络划分为区域（Area），减少路由信息的传播范围，提高网络的可扩展性和性能。
  4. 支持大规模网络：IS-IS被设计用于支持大规模网络，它使用了层次化的设计和路由聚合机制，减少路由表的规模和路由更新的开销。
  5. 支持IPv4和IPv6：IS-IS既支持IPv4网络环境，也支持IPv6网络环境，可以在不同的IP版本之间进行路由转发。
  6. 快速收敛：IS-IS具有快速的收敛性，可以迅速适应网络拓扑变化和链路状态的更新，保持网络的稳定性和可靠性。
  7. 安全性：IS-IS提供了一些安全机制，如认证和密钥管理，用于保护路由信息的安全性和完整性。

思考题：Docker与虚拟机有什么不同？

虚拟机模拟整个操作系统，而Docker只实现操作系统的一个子集，将所装载的应用程序隔离开来。后者所消耗的资源较前者更少，效率更高，在不需要完整虚拟机功能的情况下是较优选择。

思考题：Golden key

外键约束用于建立表间关系（M对N鸟足图），维护表间数据的完整性和一致性。

内键约束用于定义主键列，确保主键唯一性。

外键约束影响性能，增大数据操作阻碍，让数据库结构复杂化。

思考题：X与鲸鱼

PVE是完整虚拟化平台，每个虚拟机有自己的网络栈，独立的网络接口、IP地址和网络配置，使用Linux桥接，还支持VLAN/VXLAN等虚拟网络技术。

Docker是轻量容器化平台，容器共享主机的网络栈，直接使用主机的网络接口和IP。
